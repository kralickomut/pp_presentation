Reconciliation

Jak React hledá minimální změnu v DOM

Bez frameworku — ruční přístup

// Po každé změně musíme sami vědět,
// co přesně aktualizovat
document.getElementById('count')
  .textContent = newCount;

document.getElementById('btn')
  .disabled = newCount >= 10;

document.getElementById('status')
  .className = newCount > 5
    ? 'warn' : 'ok';
Každá změna stavu = ruční hledání správného uzlu. Chyba = špatný DOM.

React — reconciliation

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <span id="count">{count}</span>
      <button disabled={count >= 10}
        onClick={() => setCount(c => c+1)}>
        +
      </button>
      <span className={count > 5
        ? 'warn' : 'ok'} />
    </div>
  );
}
setState() → nový VDOM strom → diff se starým → React aktualizuje jen uzly, které se lišily.
